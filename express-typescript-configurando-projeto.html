Como configurar o Express com o TypeScript

MAI 16, 2020

Neste tutorial irei demonstrar como criar a estrutura de uma aplicação Express com o TypeScript

Rodrigo Dias NoronhaMAI 16, 2020

É bem comum encontrarmos pequenas aplicações Node feitas a partir do Framework Express. Express é excelente para a construção de pequenos microsserviços e possui integração com grandes libs utilizadas no mercado. Por exemplo, você pode trabalhar com serverless junto ao Firebase utilizando uma aplicação Express por debaixo dos panos. Você pode utilizá-lo para tanto como uma API Rest quanto com GraphQL. Ambos são totalmente suportados pelo Express. No entanto, o foco deste artigo será demonstrar como fazer uma configuração de projeto básica, onde você consiga construir pequenas aplicações a partir desta estrutura utilizando TypeScript.

Mas porque TypeScript? Talvez eu aborde isso em um outro artigo, mas saiba que é bastante importante ter conhecimento em TS nos dias de hoje. A maioria das javascript libraries open source estão sendo refeitas em TS. Ou seja, se o mundo open source está adotando TS pra desenvolvimento, talvez você também devesse, dependendo dos seus objetivos.

Chega de enrolação.

Vamos iniciar o projeto

  • Primeiro, vamos criar um diretório para a nossa aplicação e entrar nele.
    mkdir express-ts && cd express-ts
  • Agora, vamos iniciar o npm e instalar algumas dependências do nodso projeto.
npm init -y && npm i express

Pronto. Se tivéssemos um projeto com JavaScript sem TS, então tudo estaria terminado. Está terminado? Não. Quando trabalhamos com TypeScript em nossos projetos, precisamos, na maioria das libs, instalar as tipagens daa libs que vamps utilizar, caso contrário o próprio TypeScript reclama. Mas lembre-se, fazemos isso apenas para bibliotecas que mantém suas tipagens em pacotes no NPM separadas, então não faremos para todas, tecnicamente.

Instale a tipagem do Express e o TypeScript em si. Lembrando que o TypeScript é usado apenas para desenvolvimento, logo todas as dependências dele precisam ser colocadas como dependências de desevolvimento. Basta acrescentar o '-D' no final do comando, como no comando abaixo.

npm i @types/express typescript -D

Um macete para instalar as tipagens quando o TS reclamar seria instalar a dependência com um "@types/" antes do nome da dependência, como na dependência acima.

TS somente em desenvolvimento...

Antes de continuarmos, uma breve explicação: O TypeScript não é entendido diretamente pelo Node (a não ser pelo Deno) ou pelo interpretador dos Browsers. Ele adiciona a tipagem no desenvolvimento de qualquer aplicação e, quando for para produção, geralmente você gera o build da aplicação para usar. Mas para usarmos o TS em desenvolvimento, precisamos de um bundle que o compile, poderia ser o próprio TSC do TypeScript ou algum Babel.js da vida. No entanto, iremos utilizar o Sucrase para este projeto.

Para instalar o sucrase, execute o comando abaixo:

npm i sucrase -D

Iremos precisar também do Nodemon para nosso servidor local ficar reiniciando toda vez que editarmos algum código da nossa API. Para instalar o Nodemon, segue o comando:

npm i nodemon -D

Precisamos fazer mais alguns passos para usar metermos a mão no código.

Primeiro, vamos configurar o Nodemon para que ele utilize o Sucrase para ficar observando as mudanças no nosso código. Então crie um arquivo chamado nodemon.json na raiz do projeto e adicione o seguinte código:

{
  "watch": ["src"],
  "ext": "ts",
  "execMap": {
    "ts": "sucrase-node"
  }
}

Precisamos criar um arquivo chamado tsconfig.json na raiz do projeto também. Este arquivo contém informações importantes do TS, seja na hora de compilar quando até para desenvolvimento. Adicione o seguinte conteúdo ao arquivo (ou crie o seu próprio com sua própria configuração).

{
  "compilerOptions": {
    "module": "commonjs",
    "allowJs": true,
    "alwaysStrict": true,
    "esModuleInterop": true,
    "rootDir": "src",
    "outDir": "dist"
  },
  "exclude": ["node_modules"],
  "include": ["src"]
}

Legal. Último passo. Adicione a seguinte linha em scripts ao seu package.json:

{
  "name": "siciliana",
  "version": "1.0.0",
  "main": "index.js",
  "license": "MIT",
  "dependencies": {
    "express": "^4.17.1"
  },
  "devDependencies": {
    "@types/express": "^4.17.2",
    "nodemon": "^2.0.2",
    "sucrase": "^3.12.1",
    "typescript": "^3.7.5"
  },
  "scripts": {
    "dev": "nodemon src/index.ts" // adicione essa linha
  }
}

Agora, criamos uma pasta chamada src e um arquivo index.ts dentro dela.

Legal. Agora podemos utilizar o TypeScript com Express. Vamos criar um mini servidor dentro do arquivo src/index.ts:

import express from "express";

const server = express();

server.get("/", (request, response) => response.json({ hello: "world" }));

server.listen(3333, () => console.log(`Server running port 3333`));

Salve o arquivo e execute no terminal:

npm run dev

Abra seu navegador em localhost:3333. Teremos esta resposta.

Agora poderiamos utilizar essa estrutura para criar qualquer outro tipo de aplicação com o Express. Claro, poderia ter mais configurações extras como adicionar o ESLINT junto com o Prettier, mas ai fica como lição para vocês. Aliás, postar sobre o ESLINT ou o Prettier... Bônus: Adicionando suporte a testes no nosso projeto com Jest

Sim, se vamos fazer aplicações profissionais, em suma é importantíssimo fazermos testes nela. Principalmente se for no backend. Na minha opinião, é obrigatório uma aplicação de mercado possuir testes. Seria loucura não ter.

Então vamos lá, iremos utilizar a biblioteca do Facebook chamada Jest.

Para isso, instale o Jest globalmente:

npm i -g jest

Depois disso, execute jest --init na raiz do diretório. Será perguntando algumas coisas pelo Jest, então apenas responda com y (yes) e que você executará o Jest no Node.

Será criando um arquivo jest.config.js na raiz do seu projeto. Este arquivo contém informações importantes sobre como o Jest irá executar os testes.

Mas preste atenção. O Jest, por padrão, não entende TypeScript. Precisamos configurar isso.

Como estamos trabalhando com o Sucrase, existe um plugin que integra o Jest ao sucrase, assim o Jest entenderá TS numa boa. Então, para instalar o plugin, segue o comando:

npm i @sucrase/jest-plugin -D

Vamos até o arquivo jest.config.js agora. Encontre a linha onde existe um objeto chamado transform.

Por padrão, o meu veio assim transform:undefined. Mas vamos mudar isso para:

 "transform": {
      ".(js|jsx|ts|tsx)": "@sucrase/jest-plugin"
    },

Eu acho que o melhor jeito de aprender uma tecnologia é seguindo a documentação dos projetos e libraries, então caso não encontre ou tenha dúvidas, eis o próprio tutorial do Jest-Plugin.

Legal. Era só isso de configuração. Que tal testarmos agora? Particularmente, eu gosto de criar uma pasta na raiz do projeto chamada __tests__, então faça o mesmo. É meio que um padrão da comunidade, então é legal seguir.

Para fazer teste unitário, está tudo configurado. No entanto, vamos fazer um teste de integração, testando nossa rota raiz da API. Para fazer requisições em testes, é necessário instalar uma library chamada supertest especializada para isso:

npm i supertest && npm i @types/supertest @types/jest

Eu coloquei as tipagens também porque é sempre bom você instalar a tipagem do supertest e Jest quando for utilizar eles, pois ativa todo o intellisense do Visual Studio Code (caso esteja utilizando ele).

Mas antes de testar...

Sendo direto e rápido: você precisa exportar a API antes do nosso arquivo src/index.ts. Então deixe-o assim:

import express from "express";

const server = express();

server.get("/", (request, response) => response.json({ hello: "world" }));

server.listen(3333, () => console.log(`Server running port 3333`));

export default server;

Agora, vamos criar um arquivo dentro de __tests__ chamando app.spec.ts. Adicione o seguinte conteúdo a ele:

import App from "../src";
import request from "supertest";

it("should be able to run the application", async () => {
    const response = await request.agent(App).get("/");

    expect(response.body.hello).toEqual("world");
    expect(response.status).toEqual(200);
});

Agora, basta rodar no terminal o comando npm run test e seu teste de integração deverá ser bem sucedido.
